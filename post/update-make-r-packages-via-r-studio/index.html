Update: Make R Packages via R Studio

Nov 3, 2017 R

I just posted about making R packages based on 2014 material using roxygen2 directly. As it turns out, the R Studio menus for this are even more convenient. I wanted to leave the previous post for a non-R-Studio reference, but here’s the R Studio workflow:

Using R Studio Menus to Create a New Package

  • File -> New Project -> New Directory
  • Choose “R Package”
  • Enter the desired package name and location on your local system
  • Make sure the “Create a git repository” is checked
  • hit the Create Project button
  • Edit the ‘DESCRIPTION’ file as needed.

Recommended: Enable Roxygen for documentation

The roxygen2 packages make it much easier to include documentation with your functions. To enable Roxygen when you build your packages:

  • Tools -> Project Options -> Build Tools
  • Check “Generate documentation with Roxygen”
  • Accept defaults unless otherwise inclined.

I still like Hilary Parker’s cat_function example to demonstrate how to use enter metadata for the function’s documentation in a roxygen friendly manner:

#' A Cat Function
#'
#' This function allows you to express your love of cats.
#' @param love Do you love cats? Defaults to TRUE.
#' @keywords cats
#' @export
#' @examples
#' cat_function()
 
cat_function <- function(love=TRUE){
    if(love==TRUE){
        print("I love cats!")
    }
    else {
        print("I am not a cool person.")
    }
}

Note the default template function, hello.R, does come with a .Rd documentation file in the ‘man’ folder if you’d prefer to do it manually.

Build the Package

See R Studio for more info. Under the Build Menu or Tab, you can:

  • run a check
  • build a source package (seems to run devtools::build())
  • build a binary package

Put Package on Github

If you then host your package on Github, it becomes easy to install for your future self or any collaborators/users, but using devtools:install_github:

install_github('username','repositoryName')

Assuming you already initialized a git repository from the ‘create new package’ menu:

  • In GitHub, create a new repository with the desired name. for simplicity, don’t add any readme files or licenses at this time
  • open a shell window from within your package R project (working directory will be set by default to the package location)
  • git add -A
  • git commit -m ‘initial commit’
  • git remote add origin https://github.com/username/repoName.git
  • git push -u origin master
  • semi-optional: add a README.md file

Basic Overview for Adding Functions

  • Open the package project in RStudio
  • create new .R script in ‘R’ directory
  • enter function with metadata as demonstrated in the cat example above
  • check and/or build the package using R Studio menus
  • commit changed files
  • push to github
  • re-install package for local use (either from Github or from within the R Studio Build menu)
